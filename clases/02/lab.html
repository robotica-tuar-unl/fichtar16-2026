1️⃣ Habilitar los repositorios e instalar ros-apt-source

  • Habilitar el respositorio Universe de Ubuntu
sudo apt install software-properties-common curl -y
sudo add-apt-repository universe
  • Instalar ros-apt-source
export ROS_APT_SOURCE_VERSION=$(curl -s https://api.github.com/repos/ros-infrastructure/ros-apt-source/releases/latest | grep -F "tag_name" | awk -F'"' '{print $4}')
curl -L -o /tmp/ros2-apt-source.deb "https://github.com/ros-infrastructure/ros-apt-source/releases/download/${ROS_APT_SOURCE_VERSION}/ros2-apt-source_${ROS_APT_SOURCE_VERSION}.$(. /etc/os-release && echo ${UBUNTU_CODENAME:-${VERSION_CODENAME}})_all.deb"
sudo dpkg -i /tmp/ros2-apt-source.deb

1️⃣ Instalar Docker

  • Instalar y configurar Docker según el sistema operativo, utilizando los siguientes pasos

Windows

  1. Instalar Docker Desktop 2.0 (o superior)
  2. Si se utiliza WSL 2 en Windows, asegurarse de que WSL 2 back-end está habilitado: Click derecho en el ícono de Docker y seleccionar Configuración (o Settings). Seleccionar Utilizar motor basado en WSL 2 (o Use the WSL 2 based engine) y verificar que la distribución está habilitada en Recursos > Integración WSL (o Resources > WSL Integration).
  3. Si no se utiliza WSL 2 back-end, click derecho en el ícono de Docker, seleccionar Configuración (o Settings) y actualizar Recursos > Compartir archivos (o Resources > File Sharing) con cualquier ubicación donde se almacene el código fuente. Ver tips and tricks para solucionar problemas.

Linux

  1. Seguir las instrucciones oficiales de instalación para Docker CE/EE según la distribución. Si está utilizando Docker Compose, seguir las instrucciones de Docker Compose también.
  2. Agregar el usuario al grupo docker utilizando una terminal y ejecutando:
sudo usermod -aG docker $USER
  1. Cerrar sesión y volver a loguearse para aplicar los cambios

1.1 Configuración del entorno

Se logra ejecutando los archivos de configuración cada vez que se abre una nueva consola:

  • Configurar el entorno:
source /opt/ros/jazzy/setup.bash

Se debe configurar el entorno siempre que se abra una nueva terminal

  • Para realizar este paso automáticamente puede agregarlo a su archivo ~/.bashrc
echo "source /opt/ros/jazzy/setup.bash" >> ~/.bashrc
  • Verificar la variable de entorno: ROS_DOMAIN_ID
printenv | grep ROS

Salida esperada:

ROS_VERSION=2
ROS_PYTHON_VERSION=3
ROS_AUTOMATIC_DISCOVERY_RANGE=SUBNET
ROS_DISTRO=jazzy
  • Obtener ayuda desde la terminal:
ros2 --help

2 Emisor y receptor

2.1 Instalación de paquetes de demostración

Recuerda actualizar el índice de paquetes con sudo apt update

  • Para instalar el paquete demo-nodes-py ejecutar:
sudo apt install ros-jazzy-demo-nodes-py
  • Para ver los paquetes instalados
ros2 pkg executables
  • Para ver los ejecutables de un paquete
  $ ros2 pkg executables <nombre_paquete>
ros2 pkg executables demo_nodes_py

2.2 Nodos talker y listener

Comando para ejecutar nodos

  $ ros2 run <nombre_paquete> <nombre_ejecutable>

Para ejecutar el nodo talker del paquete demo_nodes_py

ros2 run demo_nodes_py talker

y en otra terminal para ejecutar el nodo listener

ros2 run demo_nodes_py listener

Recuerda configurar el entorno correctamente en ambas terminales

Para detener el nodo: Ctrl+C

Listar los nodos

ros2 node list

Obtener información del nodo

  $ ros2 node info <nombre_nodo>
ros2 node info /talker

2.3 Topics

Listar los topics

ros2 topic list

Puedes añadir la opción -t para ver el tipo de topic

Obtener información del topic

  $ ros2 topic info <nombre_topic>
ros2 topic info /chatter

Puedes añadir la opción -v para ver toda la información

Obtener información del tipo de topic

  $ ros2 topic type <nombre_topic>
ros2 topic type /chatter

Ver los mensajes que pasan por un topic

  $ ros2 topic echo <nombre_topic>
ros2 topic echo /chatter

Medir la frecuencia con la que se publican mensajes

  $ ros2 topic hz <nombre_topic>
ros2 topic hz /chatter

Listar los topics según el tipo de mensaje

  $ ros2 topic find <tipo_de_topic>
ros2 topic find std_msgs/msg/String

2.4 Interfaces: Mensajes

Obtener información de la interfaz de mensaje

  $ ros2 interface show <tipo_de_mensaje>
ros2 interface show std_msgs/msg/String

Publicar mensajes en un topic

  $ ros2 topic pub <nombre_del_topic>
      <nombre_del_tipo_de_mensaje> '<contenido>'
ros2 topic pub /chatter std_msgs/msg/String "{data: Mensaje de prueba}"

Puedes añadir la opción --once para publicar una sola vez

2.5 Herramientas de visualización

Ver el grafo de nodos y topics

ros2 run rqt_graph rqt_graph

Visualizar los topics y mensajes

ros2 run rqt_topic rqt_topic

En caso de no encontrar los paquetes, instalar ros-jazzy-rqt-graph y ros-jazzy-rqt-topic
