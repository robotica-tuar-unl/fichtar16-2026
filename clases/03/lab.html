1 Servicios

  • Configurar el entorno:
source /opt/ros/jazzy/setup.bash

Se debe configurar el entorno siempre que se abra una nueva terminal

1.1 Preparar el escenario

Ejecutar el nodo de demostración add_two_ints_server del paquete demo_nodes_py

TipPara ejecutar 
    $ ros2 run <nombre_paquete> <nombre_ejecutable>

1.2 Interacción con servicios

Listar todos los servicios

ros2 service list

Para también ver el tipo de servicio, añadir la opción --show-types o -t:

ros2 service list -t

Por cada nodo se pueden observar 6 servicios con la palabra parameters en el nombre. Estos servicios son los por defecto para la manipulación de parámetros.

Obtener información acerca de un servicio

  • Para obtener el tipo de un servicio:
  $ ros2 service type <nombre_servicio>
ros2 service type /add_two_ints
  • Para obtener la definición del tipo de servicio:
  $ ros2 interface show <nombre_tipo_servicio>
ros2 interface show example_interfaces/srv/AddTwoInts
  • Para obtener información del servicio en sí (clientes, servidor, etc):
  $ ros2 service info <nombre_nodo>

Invocar un servicio

  • Para enviar un requerimiento manual a un servidor
  $ ros2 service call <nombre_servicio> 
      <nombre_tipo_servicio> <argumentos>
ros2 service call /add_two_ints example_interfaces/srv/AddTwoInts "{a: 7, b: 12}"

Buscar un servicio por tipo

  $ ros2 service find <nombre_tipo_servicio>
ros2 service find example_interfaces/srv/AddTwoInts

2 Parámetros

2.1 Preparar el escenario

Ejecutar nodo demostración set_parameters_callback del paquete demo_nodes_py

2.2 Interacción con parámetros

Listar todos los parámetros de todos los nodos

ros2 param list

Por cada nodo existen parámetros por defecto como por ejemplo use_sim_time

Obtener información acerca de un parámetro

  $ ros2 param describe <nombre_nodo> <nombre_parametro>
ros2 param describe /set_parameters_callback param1

Obtener el tipo y valor actual de un parámetro

  $ ros2 param get <nombre_nodo> <nombre_parametro>
ros2 param get /set_parameters_callback param1

Establecer el valor de un parámetro

  $ ros2 param set <nombre_nodo> <nombre_parametro> <valor>
ros2 param set /set_parameters_callback param1 1.0

Obtener el valor actual de todos los parámetros de un nodo

  $ ros2 param dump <nombre_nodo>
ros2 param dump /set_parameters_callback

Puedes utilizar este comando para “descargar” todos los valores a un archivo con el operador >

Por ejemplo:

ros2 param dump /set_parameters_callback > archivo_parametros.yaml

Cargar un archivo de parámetros durante la ejecución

  $ ros2 param load <nombre_nodo> <nombre_archivo>

Los parámetros de solo lectura pueden modificarse solo en el inicio del nodo

Cargar un archivo de parámetros al momento de ejecutar nodos

  $ ros2 run <nombre_paquete> <nombre_ejecutable> --ros-args 
      --params-file <nombre_archivo>
